# 图表与编号规范

本文件定义 `paper-intake-router` 在论文工作流中处理图表时的硬规则。

## 1. 图表生成原则

当论文需要图、表时，不要只在正文里写“此处插图”。

默认流程应是：

1. 先生成 `figure-table-plan.json/.md`
2. 对于可程序化生成的图表，自动写出代码与数据模板
3. 生成图像文件、表格数据文件或表格草稿
4. 在正文中使用与计划一致的编号和引用语句

适合自动代码生成的对象：

- 柱状图
- 折线图
- 热力图
- 消融实验表
- 数据集统计表
- 模型对比表
- 系统流程图的结构化草稿

不适合自动硬生成的对象：

- 需要真实实验跑数但尚无数据的结果图
- 需要用户专有数据但数据未提供的图表
- 高度依赖人工设计美术效果的示意图

对于缺数据的情况，仍应生成：

- 数据 schema
- 占位 CSV/JSON
- 代码脚手架
- 图表标题、编号、正文引用文案

## 2. 编号规则

图表编号必须由模板决定，不允许在写作时临场乱写。

优先级：

1. 用户提供的学校/期刊/会议模板
2. `selectedLayoutTemplate`
3. 若仍无明确信息，再按默认规则回退

### 默认规则

#### 中文学位论文

- 图：`图 1-1`、`图 2-3`
- 表：`表 1-1`、`表 2-2`
- 规则：按章编号
- 正文引用：`如图 1-1 所示`、`见表 2-2`

#### 中文课程论文

- 默认仍优先按章编号；若无章节结构，可退为全局连续编号
- 图：`图 1`、`图 2`
- 表：`表 1`、`表 2`

#### 英文论文

- Figure: `Figure 1`, `Figure 2`
- Table: `Table 1`, `Table 2`
- 正文引用：`As shown in Figure 1`、`see Table 2`

## 3. 标题位置

默认约定：

- 图题在图下方
- 表题在表上方

若用户模板与此冲突，以用户模板为准。

## 4. 文件命名建议

为减少错配，图表产物文件名应稳定可追踪。

建议格式：

- `artifacts/figures/ch01-fig01-model-architecture.png`
- `artifacts/tables/ch03-tab02-ablation-study.csv`
- `artifacts/codegen/ch03-tab02-ablation-study.py`

## 5. 正文约束

正文中的图表引用必须满足：

- 编号存在于 `figure-table-plan.json`
- 标题与计划一致或只做轻微语言润色
- 不得出现正文引用了 `图 3-2`，但产物中不存在该编号
- 不得同章重复使用同一编号指向不同对象

## 6. 质量检查

交付前至少检查：

- 图表编号是否连续
- 图表标题与正文引用是否一致
- 图表文件是否真实存在
- 表格列名、单位、图例、坐标轴标签是否完整
- 若为占位数据，是否显式标注为草稿/示例
