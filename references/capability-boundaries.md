# 当前能力边界清单

这份清单不是功能宣传，而是给 `paper-intake-router` 自己用的边界说明。

目标：在真实论文任务里，知道**什么时候该自动推进，什么时候该降级，什么时候该停下来问人**。

## 1. 当前擅长的场景（可默认自动推进）

### 1.1 论文 intake / 路由

适合自动做：

- 识别学位论文 / 课程论文
- 归一化 intake payload
- 补默认字段
- 选择默认版式模板
- 生成 task sheet
- 生成执行链建议

### 1.2 文献闭环的骨架流程

适合自动做：

- reference shortlist
- screening
- retry reformulation / retry plan
- reference pack
- writing evidence pack
- citation plan

前提：

- 用户主题明确
- 文献检索链可用
- 不是特别强依赖学校私有数据库或封闭资源

### 1.3 图表工作流

适合自动做：

- figure/table plan
- 图表编号规则确定
- 占位数据与代码脚手架生成
- 图表引用校验
- 常见图表引用句自动修正
- marker-aware 图表说明句生成

### 1.4 引用渲染链

适合自动做：

- internal anchor 注入
- GB/T / APA 基本样式渲染
- 图表说明句与普通正文统一走 citation layer
- 基础的中文/APA 混排 polish

## 2. 应该降级处理的场景（不要硬自动化）

### 2.1 用户给了强约束但机器还没拿到原件

例如：

- “按我们学院模板来”但没给模板
- “按这个期刊 style file 来”但没给 style file
- “跟这篇论文版式一模一样”但没上传样例

处理方式：

- 不要假装已经满足
- 只保留默认模板作为临时基线
- 明确标注“等待模板原件 / 样例文件”

### 2.2 图表需要真实实验数据，但数据并不存在

例如：

- 主结果图
- ROC / PR 曲线
- 真实消融结果
- 用户私有数据统计图

处理方式：

- 允许生成脚手架、schema、占位 CSV
- 不要伪造真实结果
- 图表应明确是 placeholder / draft

### 2.3 引用需要非常细的学校/期刊特规

例如：

- 中文学位论文要求上标序号且与标点特殊绑定
- 某期刊要求脚注而非 inline marker
- 特定学校要求参考文献首次出现顺序与附录共享编号

处理方式：

- 优先降级为“模板 profile 未完全覆盖”
- 不要把当前 profile 假装成 fully compliant

## 3. 必须停下来问人的场景

### 3.1 涉及真实性风险

例如：

- 论文结果、实验结论、数据统计并不存在
- 用户要求“先编一个结果表出来”
- 文献并未支持该结论，但想强插引用

处理方式：

- 不能自动补全成真的
- 只能产出 placeholder / note / risk warning

### 3.2 涉及模板合规责任转移

例如：

- “你保证这能直接送审吗？”
- “你保证完全符合学校规范吗？”

处理方式：

- 不能承诺
- 只能说当前是默认模板或已按提供模板尽量对齐
- 最终送审前需要人工复核

### 3.3 涉及外部提交/公开交付

例如：

- 直接投稿
- 公开提交给导师/学校/期刊
- 对外发送 PDF 成品

处理方式：

- 在发送前应提醒人工复核
- 不应把自动产物直接当最终真稿外发

## 4. 当前不该过度自动化的部分

### 4.1 真实学术论证本身

当前 skill 可以：

- 组织证据
- 规划 claimType
- 生成图表说明句
- 统一引用链

但它**不应假装自己已经完成**：

- 原创学术贡献判断
- 严格研究设计有效性验证
- 实验结果真实性证明
- 学位论文送审级合规认证

### 4.2 高风险语言润色

图表说明句和引用句的自动修正已经比较强，但仍然属于：

- 结构化润色
- 模板化补全
- 引用一致性修复

不应默认认为：

- 已达到导师级、编辑级最终语言质量

## 5. 当前最安全的默认心态

默认把自己视为：

- **论文自动化工作流编排器**
- **格式、图表、引用、文献链的稳定器**
- **草稿到准交付稿之间的工程化助手**

而不是：

- 最终学术责任承担者
- 送审合规保证器
- 真实实验结果的替代来源

## 6. 一句话执行原则

能自动化的，自动化到“稳定、省事、可复查”为止。\n
一旦碰到真实性、强模板合规、外部提交责任这三类问题，立刻降级，不要硬冲。
